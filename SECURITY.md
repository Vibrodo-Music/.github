# Security Policy

## Supported Versions

We actively monitor and patch vulnerabilities in the following core architecture tracks:

| Version | Supported |
| ------- | --------- |
| 0.1.x   | ✅ Yes    |
| < 0.1.0 | ❌ No     |

## Reporting a Vulnerability

We take the security of our streaming architecture seriously. If you discover any vulnerability or zero-day exploit within the Vibrodo ecosystem, please do **NOT** open a public GitHub issue. 

Instead, report it directly via email to: **abhinavceovibrodo@gmail.com**

### What to Include:
* A detailed description of the vulnerability.
* Steps to reproduce or a Proof of Concept (PoC) exploit script.
* Potential impact on concurrent listeners or microservices.

We will acknowledge your report within 48 hours and provide a timeline for triage and patch deployment.
